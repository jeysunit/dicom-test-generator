# 00. プロジェクト概要

## プロジェクト名

**DICOM Test Data Generator**  
**DICOMテストデータ生成ツール**

**Repository**: `dicom-test-generator`  
バージョン: **1.1**  
最終更新: 2025年2月

---

## 目的

医療画像システム（PACS、RIS、ビューア、画像取り込みシステム等）のデバッグ・検証用に、多様なパターンのDICOMメタデータを持つテストデータを生成するツール。

**重要**: このツールの目的は「プログラムを書くこと」ではなく、「テストデータを生成すること」である。

**生成されるピクセルデータは合成データであり、実際の医療画像ではありません。臨床利用はできません。**

---

## 主要機能

### Core機能

- ✅ 病院・モダリティ別のDICOMテンプレートベースのデータ生成
- ✅ 患者マスターデータからの情報取得
- ✅ 正常データ・異常データ（安全性レベル別）の生成
- ✅ 多言語対応（日本語/ローマ字切り替え）
- ✅ 1スタディに複数シリーズ・複数画像を含むリアルなデータ構造
- ✅ 医療ビューアで表示可能なピクセルデータ生成（CTリアルモード）

### オプション機能

- 🔲 DICOM Storage SCP機能（C-STORE受信） - Phase 1.5

## 対象ユーザー

- **医療システム開発者**: DICOM取り込み機能のデバッグ
- **QAエンジニア**: 異常データを含むテストケース作成
- **医療ITベンダー**: 製品検証用ダミーデータ生成

---

## 非対象

以下は本ツールの対象外：

- ❌ 実際の医療画像生成（診断目的）
- ❌ PACS本番環境への直接投入
- ❌ 患者プライバシーデータの管理
- ❌ 医療機器認証取得

## 開発方針

### 仕様駆動開発

1. **仕様を先に固める** → 実装する
2. **各フェーズは独立動作可能**
3. **生成AIによる実装を前提**とした明確なインターフェース定義

### 段階的開発（フェーズ）

```text
Phase 0: Core Engine (最優先)
   ↓
Phase 1: CLI
   ↓
Phase 2: GUI
   ↓
Phase 1.5: Storage SCP (オプション)
```

各フェーズは **前フェーズの完成後に開始**。

### 設計原則

- **過剰設計を避ける**: 最短実装可能な構造
- **レイヤー分離**: UI → Service → Core
- **医療仕様は不変**: DICOM関連の仕様変更禁止
- **テスト容易性**: Core Engineは単体テスト可能
